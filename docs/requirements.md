# glance — 要件定義書

## 1. 背景と目的

### 1.1 課題

macOS でマークダウンファイルを確認するにはエディタを起動する必要があり、素早い内容確認ができない。

### 1.2 目的

メニューバーから即座に Markdown ファイルを書式付きでプレビューできる macOS 常駐アプリを提供する。グローバルホットキーでファイルを開くことでアプリ切り替えが不要になる。

### 1.3 対象ユーザー

- macOS で日常的に Markdown ファイルを扱う開発者・ライター
- エディタを開かずにファイル内容をすばやく確認したいユーザー

---

## 2. 機能要件

### 2.1 Markdown レンダリング（コア機能）

| ID | 要件 | 優先度 |
|----|------|--------|
| F-01 | Markdown ファイルを書式付きでプレビューウィンドウに表示 | Must |
| F-02 | 見出し (h1–h6) を適切なフォントサイズ・ウェイトで表示 | Must |
| F-03 | 太字・斜体・取り消し線・インラインコードをレンダリング | Must |
| F-04 | 順序付き/なしリスト、ネストリストを正しくレンダリング | Must |
| F-05 | フェンスドコードブロック（` ``` `）をモノスペースフォント＋背景色で表示 | Must |
| F-06 | テーブルを罫線付きで表示 | Must |
| F-07 | リンクを表示（外部ブラウザで開く） | Must |
| F-08 | 画像（相対パス）を表示 | Won't: Sandbox の file access 制約のため、相対画像パスの解決は対象外 |
| F-09 | ブロック引用（`>`）を左ボーダー付きで表示 | Must |
| F-10 | 水平線（`---`）をレンダリング | Must |
| F-11 | タスクリスト（`- [ ]` / `- [x]`）をチェックボックスで表示 | Should |

### 2.2 ダークモード対応

| ID | 要件 | 優先度 |
|----|------|--------|
| F-20 | システムのダーク/ライトモードに自動追従 | Must |
| F-21 | テキスト・背景・コードブロック・ボーダーの配色を切り替え | Must |

### 2.3 ファイルアクセス

| ID | 要件 | 優先度 |
|----|------|--------|
| F-30 | グローバルホットキー (Cmd+G) でファイルオープンダイアログを表示 | Must |
| F-31 | メニューバーアイコンのメニューから「Open Markdown File…」を選択可能 | Must |
| F-32 | 選択したファイルを新規プレビューウィンドウで表示 | Must |

### 2.4 設定画面

| ID | 要件 | 優先度 |
|----|------|--------|
| F-40 | フォントサイズの変更（小 14px / 中 16px / 大 18px） | Should |
| F-41 | コンテンツ最大幅の調整（500–900px） | Should |
| F-42 | 表示言語の切り替え（System Default / English / 日本語） | Should |
| F-43 | アップデートの自動確認設定 | Should |

### 2.5 アップデート機能

| ID | 要件 | 優先度 |
|----|------|--------|
| F-50 | Sparkle フレームワークによるアップデートチェック | Should |
| F-51 | 設定画面から手動でアップデートを確認可能 | Should |

---

## 3. 非機能要件

| ID | 要件 | 目標値 |
|----|------|--------|
| NF-01 | プレビュー表示速度 | 1MB の .md ファイルを 500ms 以内に表示 |
| NF-02 | メモリ使用量 | プレビュー時 50MB 以下 |
| NF-03 | 対応 OS | macOS 26.0 (Tahoe) 以降 |
| NF-04 | サンドボックス準拠 | App Sandbox 対応 |
| NF-05 | ファイルエンコーディング | UTF-8 |

---

## 4. 対応ファイル形式

| 拡張子 | UTType |
|--------|--------|
| `.md` | `net.daringfireball.markdown` |
| `.markdown` | `net.ia.markdown` |
| `.mdown` | `public.markdown` |
| `.mkd` | `text.markdown` |

---

## 5. 制約事項

1. **サンドボックス制限**: ユーザーが選択したファイルのみアクセス可能。
2. **JavaScript**: WKWebView 内での任意の JS 実行はサンドボックスで制限される。
3. **グローバルホットキー**: Carbon Hot Key ベースで実装し、Accessibility 権限には依存しない。

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| GFM | GitHub Flavored Markdown — テーブル・タスクリスト・打ち消し線を含む Markdown 拡張仕様 |
| Ink | JohnSundell/Ink — Pure Swift GFM パーサー |
| Sparkle | macOS アプリのアップデート配信フレームワーク |
| App Group | 同一開発者の複数ターゲット間でデータを共有する仕組み |
| Status Bar | macOS メニューバー右側のアイコン表示領域 |
